Information about the Employer

The City of Brampton’s Transit & Recreation division serves a large and diverse population, managing memberships, program registrations, and facility bookings through the PerfectMind/Xplor platform. The recreation system itself is outsourced to Xplor, meaning the City collaborates closely with an external vendor for updates, maintenance, and issue resolution. This partnership requires clear communication and thorough testing before rolling out changes to avoid service disruptions for residents.

Xplor Recreation — Vendor Partner

Insight: Working with an outsourced system taught me the importance of vendor coordination, reproducible bug reports, and automation evidence that can be shared externally — making my work useful not just for internal QA but also for Xplor’s support team when diagnosing issues.

Goals and Learning Outcomes

My goals for this work term were to build reliable automation, document repeatable workflows, support residents and staff through clear communication, and identify new opportunities for efficiency. I also aimed to grow soft skills around teamwork and cross-department collaboration.

Planned Goals

  • Job-task goals: Automate high-volume Recreation tasks (account validation, checks) and produce workflow documentation/SOPs so others can run the same steps reliably.
  • Skills to learn (benefit): Python Selenium with XML locators and targeted JavaScript execution (to handle tricky elements), structured logging/screenshots for audits, and inbox triage/prioritization. These skills help me contribute faster on future QA automation and support teams.
  • Technologies & why: Python + Selenium/WebDriver (widely used, fast iteration), CSV/Excel → dashboards (to show impact to stakeholders), and exploration of OCR for invoice processing (to reduce manual data entry for Accounting).
  • Soft goals: Improve teamwork and communication (daily inbox updates, status notes), learn to write maintainable SOPs, and practice concise updates in weekly reviews.

Reflections & Results

  • Completed: Built and ran Python Selenium scripts using XML locators; added logging to CSV and screenshots for a clear audit trail; reduced per-account checks from ~20s to ~7s.
  • Completed: Wrote workflow documentation/SOPs so others could reproduce results; created simple dashboards from CSV/Excel to communicate time savings and error trends.
  • Completed: Managed the shared support inbox, reproduced issues, escalated to the vendor when needed, and closed tickets with clear notes for the team.
  • Partially completed: Accounting invoice automation (OCR) — I analyzed the workflow and proposed an approach (OCR + parsing + validation) but full implementation wasn’t shipped due to timeline and access constraints. I left a documented plan for future development.
  • Soft skills growth: Worked closely with Phebe (manager) and teammates; improved teamwork, communication, and cross-department coordination (QA ↔ Accounting ↔ Vendor).

Job Description & Key Projects

My co-op role combined technical automation, documentation, and cross-team collaboration. The most unique aspect of my job was building and running real automation scripts in a live municipal system environment, while also helping shape processes that future staff will use. I applied programming skills from school but also learned many practical techniques on the job — especially around debugging, workflow design, and team communication. The C programming labguage is mainly taught at the Univeristy of Guelph so I became more proficient at python on my co-op term.

Automation Development: Wrote Python Selenium scripts using XML locators and occasional JavaScript execution to handle complex UI interactions. Added logging, CSV output, and screenshots for a reliable audit trail. Skills: Python (classroom + practice), Selenium/WebDriver (learned on the job), DOM inspection.
Workflow Documentation: Authored clear SOPs and visual guides for running scripts, validating results, and troubleshooting. This ensured the automations could be maintained after my term ended. Skills: Technical writing (on the job), process mapping (learned from colleagues).
Inbox & Support Management: Managed the shared support inbox, responded to resident issues, and escalated problems to the vendor when necessary — giving me experience in customer service and communication. Skills: Professional communication (on the job), prioritization, escalation protocols.
Audits & Reporting: Conducted data audits, summarized results, and presented findings during weekly reviews, helping decision-makers understand trends and gaps. Skills: Data analysis (class + job), Excel/CSV handling, stakeholder presentation.
Accounting Collaboration: Worked with the accounting team to understand invoice processing workflows and suggested an OCR-based automation script that could capture and process invoice data automatically. Skills: Workflow analysis, solution design, automation planning (learned on the job).

Conclusions

This term significantly improved my ability to combine automation engineering with business systems analysis. I learned to measure the impact of my work with metrics, communicate results visually, and deliver tools others can use after my term ends.

These skills will directly benefit me in my next co-op at McKesson Canada, where I will be working in a software development and QA automation environment. My experience with Python Selenium scripting, documentation, and cross-team collaboration will allow me to contribute to McKesson's automated testing pipelines and communicate results effectively in a large-scale enterprise setting.
